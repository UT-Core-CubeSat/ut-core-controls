ADCSCore Embedded Optimization Plan
Priority 0: Correctness Bug -- Wheel Matrix S Columns Not Normalized
In [app/adcsComputeTest/forOrbit/ADCSCore/components/core_Parameters.hpp]core_Parameters.hpp line 107:
for (int i = 0; i < 4; ++i) m.col(i).normalize();
Mat::col() returns a temporary Vec<3> by value. Calling normalize() on the temporary does nothing to m. The wheel spin-axis matrix S has un-normalized columns, which corrupts the pseudo-inverse S_pseudo and null-space projector N downstream, affecting all torque allocation and desaturation.
Fix: Use direct indexing to normalize each column in-place:
for (int i = 0; i < 4; ++i) {
    Real n = std::sqrt(m(0,i)*m(0,i) + m(1,i)*m(1,i) + m(2,i)*m(2,i));
    if (n > 1e-15f) { m(0,i) /= n; m(1,i) /= n; m(2,i) /= n; }
}
Alternatively, add a normalizeColumn(int c) method to Mat that operates on data[] directly.

Priority 1: WMM wrldmagm -- Eliminate tgamma and pow
[app/adcsComputeTest/forOrbit/ADCSCore/components/core_HelperFunctions.cpp]core_HelperFunctions.cpp lines 378-416. This function runs whenever QUEST fires (~2 Hz) and is the single most expensive function in the codebase.
1a. Replace std::tgamma with factorial recurrence (lines 385-387)
std::tgamma(n+1) computes n! using the full gamma function -- extremely costly on soft-float or even hard-float M4. Replace with a precomputed table or accumulator:
// Precomputed table (max degree 12 -> need up to 24!)
static const Scalar fact_table[25] = {1, 1, 2, 6, 24, 120, 720, ...};
// Then:
Scalar fact_n_m = fact_table[n - m];
Scalar fact_n_p_m = fact_table[n + m];
1b. Replace std::pow(a_over_r, n+2) with accumulation (line 399)
// Before loop:
Scalar ar_pwr = a_over_r * a_over_r * a_over_r;  // (a/r)^3 for n=1
// Inside loop, at end:
ar_pwr *= a_over_r;  // (a/r)^(n+2) by recurrence
1c. Cache sin(theta) outside inner loop (lines 410-411, 430-431)
std::sin(theta) is called in both the inner accumulation loop and diff_legendre. Compute once, pass as parameter.
1d. Stack usage: P[14][14] plus temporaries
wrldmagm uses ~1.5 KB of stack locals. Consider making P, g_adj, h_adj members of HelperFunctions to avoid stack pressure per call.

Priority 4: NDI Reference Model -- Consider Simpler Integrator
[app/adcsComputeTest/forOrbit/ADCSCore/components/core_ControllerNDI.cpp]core_ControllerNDI.cpp update_reference_model (lines 169-184) uses RK4, which calls reference_model_dif_eq 4 times per cycle. Each inner call does compute_BP_Toolbox (quaternion algebra, 3x3 ops, normalize, inverse).
At 30 Hz (dt ~ 33ms), Heun's method (RK2) or even semi-implicit Euler may be stable enough for the reference model dynamics, cutting the toolbox evaluations from 4 to 2 or 1 per cycle. This needs validation against the simulation to confirm acceptable tracking error.

Priority 6: Mat::col() Returns Temporary -- Add In-Place Method
The col() by-value pattern in [app/adcsComputeTest/forOrbit/ADCSCore/components/core_Math.hpp]core_Math.hpp (line 371) causes the core_Parameters.hpp bug and makes normalizeColumns() (line 409) silently broken. Add a normalizeColumn(int c) method that operates directly on data[]:
void normalizeColumn(int c) {
    Real n = 0;
    for (int i = 0; i < R; ++i) n += (*this)(i,c) * (*this)(i,c);
    n = std::sqrt(n);
    if (n > Real(1e-15)) for (int i = 0; i < R; ++i) (*this)(i,c) /= n;
}
